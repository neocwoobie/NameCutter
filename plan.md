# NameCutter `plan.md`

## Summary
- 建立一個 Windows 桌面小工具 `NameCutter`，用來批量處理指定資料夾內檔案，當「輸出後完整絕對路徑」超過 `66` 字元時，自動截短檔名，優先保留前段字元，保留原子資料夾結構。
- 專案採 `Python 3.13 + Tkinter + PyInstaller`，做成單視窗 GUI，支援「輸出到另一資料夾」與「輸出資料夾等於來源資料夾的原地更名」兩種模式。
- 版本控制以 [NameCutter](https://github.com/neocwoobie/NameCutter) 為目標倉庫；目前倉庫為空，規劃包含初始專案骨架、可重複打包的 Windows `exe` 流程，以及基本使用文件。

## Key Changes
- 初始化新專案骨架，至少包含專案設定、README、`.gitignore`、主程式入口、核心邏輯模組、GUI 模組、測試目錄、以及 Windows 打包腳本。
- 核心程式採「引擎與介面分離」：
  - `ScanOptions`：`source_dir`、`output_dir`、`max_path_length`、`in_place`
  - `PreviewItem`：`source_path`、`target_path`、`action`、`status`、`reason`
  - `ExecutionSummary`：`processed`、`changed`、`skipped`、`failed`
- 實作掃描與預覽流程：
  - 遞迴掃描來源資料夾中的檔案。
  - 先計算每個檔案在輸出位置的完整絕對路徑。
  - 若長度 `<= 66`，預設不改名。
  - 若長度 `> 66`，只截短最後一段檔名，保留副檔名，從尾端截去多餘字元。
  - 若截短後發生重名，自動在副檔名前加 `_1`、`_2` 等序號，必要時同步再縮短主檔名，直到符合 `66` 限制。
  - 若即使縮到極短檔名仍無法讓完整絕對路徑 `<= 66`，該檔案標記為 `skip`，並在原因欄與結果摘要中報告。
- 實作單視窗 GUI：
  - 來源資料夾選擇器。
  - 輸出資料夾選擇器。
  - `輸出與來源相同` 勾選框。
  - `最大路徑長度` 輸入框，預設值 `66`，允許調整。
  - `掃描預覽` 按鈕。
  - 預覽表格，顯示原路徑、新路徑、動作、狀態、原因。
  - `開始執行` 按鈕與結果摘要區。
- 執行模式規則：
  - 當 `output_dir != source_dir` 時，保留原子資料夾層級並複製輸出。
  - 當 `output_dir == source_dir` 或勾選原地模式時，執行實際更名，不搬移資料夾。
  - 預設必須先完成預覽，使用者確認後才執行。
- 打包與發行：
  - 使用 PyInstaller 產出 Windows `NameCutter.exe`。
  - 先以單一 `exe` 發行格式為主，提供一個可重複執行的 `build.ps1`。
  - README 記錄本機執行、打包、以及常見限制說明。
- Git 與倉庫落地：
  - 在本地初始化 git 並對應遠端 `https://github.com/neocwoobie/NameCutter`。
  - 至少形成「初始骨架 commit」與「可執行版本 commit」兩個清楚節點。

## Public Interfaces / Behavior
- GUI 對外行為以「先預覽、後執行」為固定主流程，不提供預設略過預覽的快捷模式。
- 路徑長度判斷依「輸出後完整絕對路徑」進行，而不是只看檔名。
- 只縮短檔名，不縮短資料夾名稱，不改變資料夾層級。
- 檔名截斷保留前綴與原副檔名；碰撞處理一律自動加序號，不覆蓋既有檔案。
- 無法壓到限制內的檔案一律跳過並報告，不做平鋪輸出，也不改動原資料夾結構。

## Test Plan
- 單元測試：
  - 路徑未超限時不改名。
  - 超限時正確截斷主檔名並保留副檔名。
  - 多個檔案截斷後重名時，正確產生 `_1`、`_2`。
  - 原地更名模式與異地複製模式都能產生正確預覽。
  - 保留子資料夾結構後仍無法壓到 `66` 時，正確標記 `skip`。
  - 中文檔名、空白、混合副檔名的行為正確。
- 手動驗證：
  - 用多層深路徑樣本測試預覽表格是否清楚顯示變更。
  - 驗證同目錄模式會只更名、不搬動資料夾。
  - 驗證異地輸出模式會保留原子資料夾並輸出到目標資料夾。
  - 驗證打包出的 `exe` 可在 Windows 直接啟動並完成一次完整流程。

## Assumptions
- `66` 的判斷單位是 Windows 端輸出絕對路徑的「字元數」，不是 UTF-8 位元組數。
- `66` 會做成可調整欄位，但預設值固定為 `66`。
- 目前範圍只處理檔案，不處理資料夾名稱截斷，不加入拖放、右鍵殼層選單、或 GitHub Actions 自動發版。
- `plan.md` 內容即以上規劃；進入實作階段後，將這份內容寫入倉庫根目錄 `plan.md`。
